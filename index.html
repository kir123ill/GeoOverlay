Свойства снимка
Папку с фото достаточно выбрать один раз в текущей сессии. Сам файл изображения используется только для привязки и предпросмотра, но не записывается в JSON. В JSON сохраняются точки, привязка, области, комментарии, метаданные, имя файла и внешний путь/URL.

2. Привязка

Точки на фото ставятся в левом окне, точки на карте — в правом. Маркеры сделаны маленькими для более точной установки.

Снимки в проекте

Краткое обучение

Режим «Редактор»

  1. Выберите папку с фото или загрузите отдельный снимок.
  2. Выберите снимок из списка, укажите название, комментарий и при необходимости внешний путь/URL.
  3. Поставьте две точки на фото и две соответствующие точки на карте.
  4. Нажмите «Применить», затем нарисуйте одну или несколько областей снимка и сохраните их.
  5. Скачайте JSON. В него сохраняются привязки и метаданные, но не само overlay-фото.

Режим «Просмотр»

  1. Переключитесь на «Готовая привязка».
  2. Нарисуйте проверочную область и нажмите «Проверить».
  3. Программа загрузит нужные JSON-квадраты из папки data/tiles и покажет найденные пересечения.
  4. При выборе найденной области справа откроется снимок: красным показана сохранённая область, зелёным — проверочная область.

Структура для GitHub Pages

В корне сайта должен лежать index.html, а готовые привязки — в data/index.json и data/tiles/*.json. Большие фото лучше хранить отдельно и указывать в JSON через imageUrl. Тогда сайт подгружает только выбранное фото, а не весь архив.
